1.概述
  本文档用于描述实时数据库（RTDB）的数据模型与模型路径，以及如何使用客户端API访问RTDB管理的模型数据。

2.RTDB的数据模型与路径
  RTDB管理两类模型：设备点模型和表模型。
  路径是数据模型在RTDB中的位置索引。
   
2.1.设备点模型
  设备点模型对应ISCS配置数据库中的以下表：
  1) 设备表(dev_tbl)
     用与构建RTDB的设备模型节点。
      
  2) 离散输出点表(do_tbl)
     用于构建RTDB数据点模型节点的离散输出点节点。
      
  3) 模拟输出点表(ao_tbl)
     用于构建RTDB数据点模型节点的模拟输出点节点。
      
  4) 累积量输入点表(acc_tbl)
     用于构建RTDB数据点模型节点的累积量输入点节点。
      
  5) 离散输入点表(di_tbl)
     用于构建RTDB数据点模型节点的离散输入点节点。
   
  6) 模拟输入点表(ai_tbl)
     用于构建RTDB数据点模型节点的模拟输入点节点。
      
2.2.表模型
  表模型对应ISCS配置数据库中除了设备点模型之外的所有表。
   
2.3.实时数据库中的路径（path）
  路径是模型在RTDB中的位置索引。是由标签（节点标签、属性/字段标签）和连接符（':'、'.'）构成的字符串。
  路径的通用格式如下：
  "域节点标签[:车站节点标签][:专业系统节点标签][:设备节点标签][:数据（信号）点节点标签][.属性标签][.子属性标签]"
  
  路径构成部分说明：
  1) 域节点标签
     路径的域节点部分，值为domain_tbl表的domain_label字段对应的值。如"occ"、"njnz"等。
   
  2) 车站节点标签
     路径的车站节点部分，值为station_tbl表的station_label字段对应的值。例如"occ"，"njnz"等。

  3) 专业系统节点标签
     路径的专业系统节点部分，值为pro_system_tbl表的pro_system_label字段对应的值。例如"bas"，“pis”等。
   
  4) 设备节点标签
     路径的设备节点部分，值为dev_tbl表的dev_label字段值的设备部分。
   
  5) 数据点节点标签
     路径的数据点节点部分，值为个点表的xxx_label字段值的数据点部分。
   
  6) 属性标签
     路径的属性部分，值为字段名称或表名称。
      
  7) 子属性标签
     路径的属性部分，是属性的子属性，值为字段名称。
      
  8) ':'
     路径的连接符，用于连接节点节点标签。
      
  9) '.'
     路径的连接符，用于连接节点标签与属性标签，或属性标签与属性标签。
      
3.模型数据访问
  模型属性分类：直接属性和引用属性。
  
  直接属性。直接属性是节点自己管理维护的属性。直接属性的访问只能通过属性所在节点路径去访问。例如通过设备节点只能访问设备节点自己的直接属性，
  不能通过设备节点去访问专业系统的直接属性。直接属性的访问路径中只有一级属性标签。直接属性的访问路径示例：
  "节点标签.属性标签"
  "节点标签：节点标签.属性标签"
  
  引用属性。引用属性是别的节点管理维护的属性，本节点中仅地址引用。可以通过本级节点访问本级节点所有的引用属性，可以访问本级节点的所有上级节点的引用属性。
  引用属性的访问路径中有两级属性标签。引用属性的访问路径示例：
  "节点标签.属性标签.子属性标签"
  "节点标签:节点标签.属性标签.子属性标签"

3.1.表模型属性的访问
  可使用的接口：
    rtdb_client_query_table
    rtdb_client_delete_records
    rtdb_client_update_record
    
   表模型数据的访问，直接使用表名称+字段名称的方式去访问。
